# 06. System Design and Architecture

System design helps you think beyond code and build reliable software.

## Core Concepts

- Scalability
- Availability
- Reliability
- Latency vs throughput
- Caching and queues

## Recommended Resources

- System Design Primer: https://github.com/donnemartin/system-design-primer
- ByteByteGo blog: https://blog.bytebytego.com/
- Martin Fowler: https://martinfowler.com/
- AWS architecture center: https://aws.amazon.com/architecture/

## Topics To Learn in Order

1. Client-server basics
2. Load balancing
3. Caching layers
4. Database scaling
5. Message queues
6. Monitoring and observability

## Practice Approach

- Pick one real app (chat, ride-sharing, LMS)
- Draw architecture diagram
- Identify bottlenecks
- Propose scaling strategy

## Output to Publish

- One markdown case study per system
- One architecture diagram per case
- Trade-off analysis section
