# 12. Prompt Templates for Better Results

Good prompts produce better technical output.

## Feature Build Prompt

Context: I am building [project] with [tech stack].
Goal: Implement [feature].
Constraints: Keep beginner-friendly, modular, and production-safe.
Output: Step-by-step changes, then final code.
Validation: Include edge cases and test scenarios.

## Debugging Prompt

Context: Here is my error and code snippet.
Goal: Find root cause and fix.
Constraints: Explain why error occurs.
Output: Minimal patch and prevention tips.

## Refactor Prompt

Context: Here is working code.
Goal: Improve readability and maintainability.
Constraints: Do not change external behavior.
Output: Refactored version + explanation of improvements.

## Documentation Prompt

Context: Here is project structure and features.
Goal: Generate professional README.
Constraints: Keep concise and actionable.
Output: Overview, setup, features, troubleshooting.
