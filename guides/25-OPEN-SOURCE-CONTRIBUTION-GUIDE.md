Use this guide to move from learning alone to collaborating in public repositories.

## First Contribution Workflow

1. Pick a beginner-friendly repository
2. Read contribution rules and setup docs
3. Find a small issue (docs, tests, bugfix)
4. Fork, branch, implement, and open PR

## Good First PR Types

- Documentation improvements
- Typo and clarity fixes
- Test coverage for existing logic
- Small UI bug fixes

## Pull Request Checklist

- Clear PR title
- Problem and solution summary
- Screenshots or logs when needed
- No unrelated changes
- Respect project lint/test rules

## Collaboration Etiquette

- Be polite and concise in review replies
- Ask questions with context
- Accept feedback and iterate quickly
- Thank maintainers

## Contribution Tracker

- Repo name

- PR link
