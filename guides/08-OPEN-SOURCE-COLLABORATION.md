# 08. Open Source and Collaboration

Open source participation builds engineering communication and teamwork skills.

## Starter Platforms

- First Contributions: https://github.com/firstcontributions/first-contributions
- Up For Grabs: https://up-for-grabs.net/
- Good First Issue: https://goodfirstissue.dev/
- GitHub Docs: https://docs.github.com/

## Collaboration Standards

- Use clear issue titles
- Keep PRs small and focused
- Add screenshots or logs when useful
- Respect code style and contribution guide

## Commit Message Quality

- Conventional Commits: https://www.conventionalcommits.org/

Examples:
- `feat: add week-4 project checklist`
- `fix: correct SQL join query example`
- `docs: improve setup section`

## First PR Checklist

- [ ] Forked repository
- [ ] Created feature branch
- [ ] Added meaningful commit messages
- [ ] Tested changes locally
- [ ] Wrote clear PR description
