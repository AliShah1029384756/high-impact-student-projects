# Contributing Guide

Thanks for your interest in contributing.

## You Can Contribute By

- Adding new high-impact project ideas
- Improving MVP definitions
- Suggesting better tech stack choices
- Fixing structure, grammar, and docs quality

## Idea Submission Format

Please include:

1. Project title
2. Problem statement
3. Target users
4. MVP feature list (3-6 points)
5. Suggested stack
6. Expected impact

## Quality Rules

- Keep ideas practical and student-friendly
- Avoid vague or overly broad project descriptions
- Prefer real problems over clone apps
- Keep writing clear and concise
