# Student Resource Guide (Extended)

This handbook is designed for CS students, graduates, and working learners across beginner, intermediate, and advanced levels.
It combines learning resources, roadmaps, career preparation, open source contribution guidance, and practical AI usage.

## How To Use This Handbook

1. Start with the level map in this file.
2. Pick one track (web, backend, data, systems, or interview prep).
3. Follow the weekly workflow at the end.
4. Use the project validation checklist before publishing work.

## 1. Level-Based Roadmaps

### Beginner Level (0-3 months)

Goals:
- Learn one programming language deeply (JavaScript or Python).
- Build 3 small projects.
- Learn Git and GitHub basics.

Recommended sequence:
1. HTML and CSS basics
2. JavaScript fundamentals
3. DOM and events
4. One CRUD mini project

Resources:
- CS50: https://cs50.harvard.edu/x/
- freeCodeCamp: https://www.freecodecamp.org/
- The Odin Project: https://www.theodinproject.com/
- MDN Learn Web Dev: https://developer.mozilla.org/en-US/docs/Learn_web_development

### Intermediate Level (3-9 months)

Goals:
- Build full projects with frontend + backend.
- Use APIs and databases.
- Improve debugging and code organization.

Resources:
- Full Stack Open: https://fullstackopen.com/en/
- Node.js docs: https://nodejs.org/en/docs
- Express docs: https://expressjs.com/
- PostgreSQL docs: https://www.postgresql.org/docs/

### Advanced Level (9+ months)

Goals:
- Design scalable systems.
- Add testing, CI/CD, and deployment.
- Build interview-ready depth.

Resources:
- System Design Primer: https://github.com/donnemartin/system-design-primer
- Kubernetes docs: https://kubernetes.io/docs/home/
- Docker docs: https://docs.docker.com/
- AWS Architecture Center: https://aws.amazon.com/architecture/

## 14. Portfolio Quality Rubric

Score each project from 1 to 5 on:
- Problem clarity
- Code quality
- Documentation quality
- Real-world usefulness
- UI/UX and accessibility
- Deployment and reliability

Target: average score >= 4.

## 17. Weekly Execution Framework

Use this recurring cycle:

1. Plan
- Pick one micro-goal for coding
- Pick one micro-goal for CS theory

2. Build
- Implement feature in small commits

3. Validate
- Test edge cases
- Lint and format

4. Document
- Update README and changelog

5. Share
- Push to GitHub
- Ask for feedback
